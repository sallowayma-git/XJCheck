## Evidence synthesis

The strongest and most consistent vendor evidence is that mature ECAD systems do
not audit by re-running local visual proximity rules over pages. They maintain
project-level connection, component, terminal, wire-number and cross-reference
objects; update that model; then run configurable checks and issue workflows.
Because the supplied DWGs are downstream drawing artifacts rather than native
ECAD projects, XJCheck must reconstruct the minimum necessary object model from
vector geometry, blocks, attributes, text and sidecars.

The strongest implementation conclusion is therefore not "replace rules with a
GNN." It is:

```text
CAD primitives
-> auditable topology decisions
-> verified symbol ports/internal connectivity
-> electrical networks
-> semantic attachments
-> project-level endpoint identities
-> constraint resolution
-> rule checks
```

Learning is justified only for ambiguous classification/ranking after this graph
exists and after labels are split by held-out project.
