# 符号库与语义恢复

## 1. 符号库的定位

符号库不是为了“识别所有电气元件型号”，而是为了回答连接审计所需的最小问题：

- 一个块是不是电气连接对象；
- 它有几个端口；
- 端口在哪里；
- 端口方向是什么；
- 内部哪些端口永久导通、条件导通或互相隔离；
- 哪些文本是端子号、设备名或说明；
- 旋转、镜像、缩放后如何映射。

## 2. 三层符号来源

### 企业/项目符号库

优先级最高。直接从当前 502 张图的 block definitions 和人工裁决建立。

### 标准符号参考

IEC 60617、GB/T 4728 等用于 taxonomy 和命名参考。标准图形或数据库可能有许可限制，不应未经授权打包复制；本项目主要借鉴分类体系和端口语义。

### 开源符号库参考

QElectroTech、KiCad 等可用于研究 schema、分类、别名和图形指纹，但不能假设其符号与当前 SuperWORKS/企业块一一对应。

## 3. Bootstrap 流程

```text
blocks.parquet
-> normalize block names
-> group by definition hash
-> count instances/projects/pages
-> collect transforms
-> collect external line entry clusters
-> collect ATTRIB/text slots
-> rank by frequency × audit impact × unknown rate
-> human annotate top families
-> write symbol_library.yml
```

## 4. 几何指纹

推荐字段：

- entity type histogram；
- line/arc/circle/polyline count；
- normalized bbox aspect ratio；
- endpoint degree histogram；
- internal graph Weisfeiler-Lehman hash；
- text/attribute tags；
- symmetry；
- port-entry clusters。

block name 只是一个特征，不能作为唯一识别依据。

## 5. 端口推断

已知 family：从 local port 直接变换。

未知 family：

1. 找所有外部线与 block bbox 的交汇；
2. 聚类入口位置；
3. 计算局部边界参数 `side + normalized offset`；
4. 与同 definition 的多个实例对齐；
5. 形成 port candidates；
6. 人工确认后升级为 family definition。

## 6. 内部连通性

端口关系枚举：

- `permanent_connected`；
- `isolated`；
- `conditional`；
- `switch_contact`；
- `pass_through`；
- `visual_only`；
- `unknown`。

MVP 只自动使用 `permanent_connected` 和 `isolated`。其他类型不做功能仿真，保持 boundary 或 review。

## 7. 版本与治理

每个 family 必须记录：

- version；
- author/reviewer；
- source projects；
- verified instances；
- last validation date；
- regression fixtures；
- deprecation state。

修改端口坐标或内部连通性必须触发相关项目 regression。
