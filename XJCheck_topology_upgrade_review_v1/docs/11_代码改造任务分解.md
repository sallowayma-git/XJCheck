# 代码改造任务分解

本文件可直接转成 GitHub Issues/Epics。

## Epic 0：Baseline Freeze

### T0.1 全语料运行器

新增：`tools/run_corpus_baseline.py` 或使用本包脚本。  
输入：数据根、仓库根、输出根、ODA 路径。  
输出：每项目 exit code、耗时、页数、转换状态、实体数、Pair 数、Issue 数。

验收：27 个项目全部出现在 run summary；失败不可被吞掉。

### T0.2 Legacy Golden

冻结：

- manifest；
- pages/texts/lines/blocks；
- line_groups/pairs/issues；
- coverage；
- topology shadow；
- 运行配置和 Git commit。

### T0.3 项目级 split

将 `reports/benchmark_split_proposal.csv` 复制到仓库 test assets，并人工确认最终 train/validation/test。

## Epic 1：Reader Adapter

### T1.1 新建 `src/dwg_audit/readers/base.py`

定义 `CadReader`, `ReaderProbe`, `ReaderOptions`, `CadDocument`, `ReaderError`。

### T1.2 新建 `readers/registry.py`

按优先级探测后端，记录 capability：

```text
native_dwg
blocks
xdata
layouts
preview
write_support
```

### T1.3 改造 ODA

从 `ingest/dwg_converter.py` 移出路径探测。支持：

- Windows executable；
- `shutil.which`；
- Linux AppImage；
- `ezdxf.options` unix path；
- xvfb/headless；
- health check。

### T1.4 Incomplete extraction gate

若 audit-required 页面实体为 0，则项目状态为 incomplete，RuleEngine 不得输出 clean conclusion。

## Epic 2：Primitive Model

### T2.1 新建 `topology/entity_normalizer.py`

处理 LINE/LWPOLYLINE/POLYLINE/ARC/CIRCLE/INSERT，生成 primitive rows。

### T2.2 新建 `topology/block_transform.py`

支持 rotation、mirror、non-uniform scale、nested block path。

### T2.3 Provenance

所有 primitive 保存 entity handle、parent handle、definition、layout 和 reader backend。

## Epic 3：Geometry Graph

### T3.1 `spatial_index.py`

为 endpoint/segment/block/text 建 STRtree。

### T3.2 `intersection_builder.py`

生成 endpoint-endpoint、endpoint-on-segment、segment intersection、collinear overlap 候选。

### T3.3 `segment_splitter.py`

只在 ASSERTED junction 处切分，保存 parent/child 映射。

### T3.4 `junction_classifier.py`

实现四态分类和 reason codes。初版只允许强确定性规则 ASSERT。

### T3.5 `gap_candidates.py`

把现有 inline text/block span bridge 改成 POSSIBLE candidate，不调用 union。

### T3.6 `geometry_graph.py`

输出 nodes/edges/decisions，不输出 Pair。

## Epic 4：Symbol Registry

### T4.1 Schema/loader

新增 `symbols/schema.py`, `symbols/registry.py`，加载 `configs/symbols/*.yml`。

### T4.2 Fingerprint

新增 `symbols/fingerprint.py`，按 block definition 建几何指纹。

### T4.3 Instance/port transform

新增 `instance_resolver.py`, `port_transform.py`。

### T4.4 Port binder

新增 `port_binder.py`，用外部线—端口几何生成 candidates。

### T4.5 迁移硬编码

把 candidates/component/wire_components 中 block name/family 判断迁入 YAML，要求迁移前后 legacy 结果零漂移。

## Epic 5：Network Builder

### T5.1 `net_builder.py`

只对 ASSERTED 边构建 connected components。

### T5.2 `open_endpoints.py`

区分：

- true open；
- symbol port；
- cross-page interruption；
- possible boundary；
- drawing boundary artifact。

### T5.3 `network_validator.py`

输出 overmerge/split suspicion，不直接改网。

### T5.4 `path_finder.py`

为 Issue 生成最短 witness path。

## Epic 6：Semantic Layer

### T6.1 `token_parser.py`

解析 prefix/local/full terminal/wire/device/page reference。

### T6.2 `project_profile.py`

消费 `.prj` 和 `LdDzbInfo.xml`，输出 terminal vocabulary 和 page priors。

### T6.3 `text_role.py`

统一现有 semantic-row/body-port/scoped-prefix 角色。

### T6.4 `text_attachment.py`

生成 top-k attachment candidates；邻域只在这里使用。

### T6.5 `scope_resolver.py`

把当前散落 scope 规则收敛到统一作用域对象。

## Epic 7：Constraint Resolver

### T7.1 OR-Tools 依赖

增加 optional dependency `ortools`。

### T7.2 `constraint_resolver.py`

实现 port/net、text/endpoint、cross-page candidate 的联合选择。

### T7.3 Alternative solution

保存 second-best gap；近似多解进入 review。

## Epic 8：Cross-page Graph

### T8.1 `terminal_identity.py`

canonical identity，不再只比较裸数字。

### T8.2 `cross_page_reference.py`

生成候选、reciprocal 和 direction。

### T8.3 `project_graph.py`

节点为 endpoint/identity/sheet，边为 selected/candidate/conflict。

## Epic 9：Audit V2

### T9.1 Rule API

Rule 输入只接收 ProjectGraph/Findings V2。

### T9.2 新规则

实现主任务书第 11 节规则。

### T9.3 Witness

每个 issue 生成完整 path 和 weakest evidence。

### T9.4 Legacy comparison

同一 run 输出 legacy/new issue 差异，不立即删除 legacy。

## Epic 10：Page Strategy

### T10.1 多标签分类

将 page classifier 从唯一 route 改为 capability labels。

### T10.2 Backplate audit

背板不再默认 classify-only；进入 symbol/table hybrid。

### T10.3 Communication medium

新增连接媒介，避免通信线与 electrical wire 混同。

### T10.4 Unknown page

输出 review profile，不允许新文件名直接映射新代码分支。
