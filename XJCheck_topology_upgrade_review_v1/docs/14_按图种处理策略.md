# 按图种处理策略

## 1. 核心原则

不再为每个标题建立独立识别器，而是为页面声明所需能力组合。

```text
WireTopology
SymbolPorts
TerminalGrid
TableMapping
CrossPageReference
CommunicationMedium
MetadataOnly
```

## 2. 页面能力矩阵

| 图种 | WireTopology | SymbolPorts | TerminalGrid | CrossPage | 说明 |
|---|---:|---:|---:|---:|---|
| 普通回路 | 高 | 中 | 低 | 高 | 拓扑优先 |
| 元件接线 | 中 | 高 | 可选 | 中 | 符号端口优先 |
| 背板接线 | 中 | 高 | 中 | 中 | hybrid，不可 LayoutOnly |
| 端子图 | 低/中 | 中 | 高 | 高 | 表格/身份优先 |
| 端子表 | 低 | 低/中 | 高 | 高 | 表格线非导线 |
| 网络通信 | 中 | 高 | 低 | 中 | medium-aware connector graph |
| 屏面布置 | 低 | 低 | 低 | 低 | metadata only |

## 3. 第三组新增结构处理

### 交换机/通信管理机/防火墙

- 设备块作为 symbol；
- 端口作为 port；
- 线标记 medium；
- 端口号/接口名附着 port；
- 不按普通水平线两端数字生成 Pair。

### PMU 背板

- block/port family；
- 多排端口；
- 局部编号；
- 背板与原理页 cross-reference；
- unknown family 自动降级 review。

### 操作箱原理

- 普通回路 topology；
- 大量符号遮挡 gap；
- 必须依赖 port/internal connection；
- 不能因文字/块在 gap 内直接 merge。

### 电度表接线

- symbol/terminal hybrid；
- 可能有电流/电压多通道；
- 端口和编号的行列关系重要。

### 时钟对时

- connector graph；
- medium 为 time_sync/serial/fiber；
- 规则重点是接口身份和对应关系，不是普通电气导通。

### 端子表

- 先检测 cell/grid；
- 建 row/column/header scope；
- 生成 terminal mapping；
- 局部线仅补充，不构成整页主网络。
