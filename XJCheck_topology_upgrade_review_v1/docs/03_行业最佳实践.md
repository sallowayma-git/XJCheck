# 行业最佳实践

## 1. AutoCAD Electrical

官方 Electrical Audit 检查对象包括未连接线端、缺失或重复线号、重复组件/端子/引脚、电缆和颜色、带连接属性但没有导线的组件，以及同一 wire network 上类别冲突。错误可以按项目/当前图显示、忽略或标记为 Issue、定位并导出。

更重要的是，wire gap 两侧不是靠再次搜索邻域恢复，而是维护 invisible Xdata pointers；跨页 signal 使用命名 source/destination code，同名 code 把不同图中的网络视为同一连续信号，并同步线号。

对 XJCheck 的启示：

- Gap 必须是显式关系对象；
- 跨页端点必须有身份，而不是只按两个数字全局查重；
- 网络是一级对象；
- 错误状态与定位是产品的一部分；
- 白名单/排除规则应项目化，而不是写死。

## 2. EPLAN

EPLAN 的项目检查首先维护 connection data，然后按 scheme 运行。检查可设为 No、Offline、Online/Offline、Prevent errors。它检查设备定义不完整、重复或缺失对象、短路、环路、缺失 cross-reference/interruption point、potential 定义等；消息数据库保存原因和解决建议。

对 XJCheck 的启示：

- 审计必须建立在统一刷新后的连接模型上；
- Check scheme 应配置化、版本化；
- “阻止错误”和“事后报告”是不同策略；
- 消息必须有原因、对象、页面、位置和解决建议；
- interruption point 应有 counterpiece 和 reciprocal rule。

## 3. Zuken E3.series

E3.schematic 强调 object-oriented project data、intelligent component library、automatic connections、logic/electric checks、online terminal plan 和 signal cross-reference。端子图与原理图变化同步，连接模式在元件移动时更新。

对 XJCheck 的启示：

- 符号库不仅存图形，还存端口、连接和制造语义；
- 端子页不是独立图像，而是同一底层连接模型的另一视图；
- 连接关系应与图形位置解耦；
- 项目级 object model 是报告和校验的单一事实源。

## 4. Siemens Capital

Capital Logic Designer 同时表达 logical connectivity 和 physical wiring，支持 topology、多抽象层和 correct-by-construction DRC。它强调统一模型、规则驱动、跨视图导航和变更一致性。

对 XJCheck 的启示：

- 区分逻辑身份、物理导线、图面位置；
- 不同页面是同一项目模型的投影；
- topology、wire、signal 和 component 不能压缩成一个 LineGroup；
- 规则运行在模型上，图形只是证据和视图。

## 5. 从成熟产品反推 XJCheck 的最小 ECAD 模型

即使只有普通 DWG，也至少要反建：

```text
Sheet
SymbolInstance
SymbolPort
PrimitiveSegment
Junction
ElectricalNetwork
TerminalIdentity
CrossPageEndpoint
CrossPageMatch
Issue
```

不需要一开始恢复完整设备功能，但必须恢复连接所需的身份和拓扑。

## 6. 矢量技术图研究

VectorGraphNET 的路线是先把矢量图转换为包含几何关系的图，再用图注意力模型做 line-level segmentation；其研究强调矢量方法相对光栅视觉方法更可扩展。工程图 GCN 和 P&ID 图结构研究也都把“构图、对象/关系分类、全图结构”作为核心。

对 XJCheck 的启示：

- 原始 DWG/DXF 不应先光栅化再从头猜；
- GNN 适合在候选图上分类，不适合替代确定性几何；
- 训练目标要明确为 segment role、junction、symbol、attachment 或 edge；
- 评估必须使用 graph-level 和 project-held-out ground truth。

## 7. 神经符号最佳实践

电气控制柜合规研究表明，在训练数据有限的工业场景中，深度学习负责感知/候选，符号逻辑负责最终合规判断是一条合理路线。

XJCheck 应采用：

```text
Deterministic candidate generation
+ probabilistic ranking
+ hard domain constraints
+ explainable rules
```

而不是单一端到端黑盒。
