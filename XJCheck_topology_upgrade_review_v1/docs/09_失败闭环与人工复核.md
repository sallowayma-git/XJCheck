# 失败闭环与人工复核

## 1. 目的

面对第三组不断出现的新结构，系统不能只靠开发者阅读 Issue 后写规则。必须把失败变成结构化数据和可复现任务。

## 2. 每页质量门

每页生成：

- reader_status；
- primitive counts；
- page structure labels；
- topology stats；
- symbol coverage；
- endpoint coverage；
- text attachment coverage；
- cross-page candidate coverage；
- unresolved counts。

任何关键指标为 0 或异常时进入 failure queue。

## 3. Failure Queue 字段

```text
failure_id
project_id
sheet_id
filename
failure_family
severity
primary_object_id
bbox
legacy_result
new_result
evidence_refs
recommended_review
status
review_label
reviewer
```

## 4. 触发条件示例

- 页面被标为二次原理图但 line_count=0；
- 背板页 route=LayoutOnly；
- high-impact block unknown；
- network 覆盖图面超过异常阈值；
- 大量 open endpoints；
- 同一个 port 绑定多个 net；
- 数字很多但 endpoint attachment 很少；
- legacy 有 Pair、new 无 endpoint；
- new 产生 hard issue 但存在 possible topology edge；
- 同一 cross-page endpoint 有多个近似候选。

## 5. 人工标签

标签应针对具体决策，而不是只写“识别错了”：

```text
CONNECTED
NOT_CONNECTED
SYMBOL_PORT
SYMBOL_INTERNAL_PASS
SYMBOL_INTERNAL_ISOLATED
TEXT_LABELS_PORT
TEXT_LABELS_NET
TEXT_ANNOTATION
CROSS_PAGE_MATCH
CROSS_PAGE_NOT_MATCH
PAGE_NOT_AUDITABLE
```

## 6. 反馈落地

每个标签必须指向一个知识目标：

- topology rule；
- symbol library；
- scope rule；
- constraint；
- model dataset；
- project profile；
- false positive whitelist。

不得只在 issue ID 上打补丁。

## 7. 本包脚本

- `scripts/build_failure_queue.py`：从 findings/issue 产物汇总失败；
- `scripts/compare_engines.py`：比较 legacy 和 topology V2；
- `scripts/bootstrap_symbol_library.py`：把 unknown/high-frequency blocks 转成标注 backlog；
- `scripts/run_corpus_baseline.py`：逐项目运行 current-head。
