# 现有代码审查与根因

## 1. 当前主链

```text
scan
-> convert DWG to DXF
-> extract texts/lines/blocks/polylines
-> classify page
-> route by page type
-> page-specific extractor
-> LineGroup
-> TerminalCandidate
-> PairCandidate/Pair
-> RuleEngine
-> report stage topology shadow
```

主链方向导致 topology 无法纠正 PairBuilder 的基本假设。

## 2. 规则膨胀的表现

任务计划和 findings 中已经出现大量结构专用名词：

- short bridge strip；
- semantic-row local numeric；
- continuation channel；
- bridge mapping；
- semantic mapping；
- first-prefixed family；
- scoped visible prefix；
- body-port；
- complementary half-chain；
- row-band aggregation；
- KLP/ZKK/PWF 等 family。

这些知识本身有价值，但应迁移到：

- Symbol Library；
- ScopeResolver；
- Candidate feature；
- Constraint；
- Rule scheme。

继续散落在页面 Extractor 中会增加隐式耦合。

## 3. `wire_topology.py` 的技术债

### 3.1 优点

- 已从原始 line 级构图；
- 已支持 T/十字观察；
- 已有 open endpoint；
- 已能把 network 与 LineGroup 关联；
- 已有 shadow 报告和 branch-local proposal。

### 3.2 缺口

- Union-Find 只表达“已合并”，不能表达不确定性；
- `inline_text_bridge` 和 `block_span_bridge` 可直接触发 union；
- block bridge 主要基于 insertion point，不基于真实端口；
- topology 内含 KLP/CLP/ZKK 等文本角色正则；
- 长线未完全以所有交点切分为 atomic edge；
- 十字交点缺少 dot/jumper/symbol semantics 的统一分类器；
- net 没有独立 decision trace；
- topology 后于 Pair 运行。

## 4. 页面分类问题

当前分类是单标签、阈值式：

- horizontal/vertical ratio；
- polyline count；
- grid band；
- sidecar category。

建议升级为多标签结构描述：

```text
has_wire_network
has_symbol_ports
has_terminal_grid
has_table_mapping
has_backplate_layout
has_communication_medium
requires_cross_page_audit
```

同一页可以同时具备多个能力，不应只能选择唯一 Extractor。

## 5. Reader 问题

`dwg_converter.py` 固定探测：

```text
C:\Program Files\ODA\ODAFileConverter 27.1.0\...
C:\Program Files\ODA\ODAFileConverter 26.9.0\...
```

而 ODA/ezdxf 官方均支持 Linux/AppImage/`unix_exec_path`。应把转换后端抽象为 Reader Adapter，不应由 ingest 文件直接绑定外部程序路径。

## 6. 数据模型问题

当前 Pair 是主要业务对象，但 Pair 之前缺少稳定的：

- atomic segment；
- topology decision；
- symbol port；
- asserted network；
- endpoint identity。

新架构中 Pair 是项目图上的派生关系，不是页面几何的第一层真值。

## 7. 下线原则

“下线邻域逻辑”不能理解为删除所有距离计算。正确处置：

| 用途 | 是否保留 |
|---|---|
| 决定两个线段是否连通 | 否 |
| 决定最终 Pair | 否 |
| 生成文本候选 | 是 |
| 生成 port-binding 候选 | 是 |
| 作为模型特征 | 是 |
| legacy shadow | 是 |
| 与新引擎差异回归 | 是 |
