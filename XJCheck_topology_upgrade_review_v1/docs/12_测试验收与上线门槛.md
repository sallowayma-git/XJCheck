# 测试、验收与上线门槛

## 1. 测试金字塔

### 单元

几何、变换、token、约束和规则。

### 合成 DXF 集成

每个关键结构一个最小图，结果完全可控。

### 真实页 Golden

从 502 页中选择不同结构族，保存人工真值和局部裁剪。

### 项目级 held-out

最终评估只能看从未参与规则、符号库和模型调参的项目。

## 2. 需要标注的真值

- segment role；
- junction state；
- network component；
- symbol family；
- symbol ports；
- text role；
- text attachment；
- endpoint identity；
- cross-page match；
- final issue。

## 3. 拓扑指标

### Junction

```text
precision = correct asserted junction / all asserted junction
recall    = correct asserted junction / all true junction
```

### Network

- overmerge：两个真网络被错误合并；
- split：一个真网络被错误拆开；
- pairwise connectivity F1；
- open endpoint precision/recall。

对电力审计，overmerge 比 split 更危险，因为它可能制造虚假跨页一致性或隐藏错误。

## 4. 审计指标

- hard issue precision；
- hard issue recall；
- review queue precision；
- false clean rate；
- issue witness completeness；
- unresolved pages；
- per-project variance。

## 5. 推荐门槛

| 指标 | 门槛 |
|---|---:|
| audit-required 页面 Reader 完整率 | 100% 或明确 incomplete |
| asserted crossing false connect | 接近 0 |
| hard issue precision | ≥99% |
| hard issue witness 完整率 | 100% |
| unresolved topology 产生 critical | 0 |
| unknown high-impact symbol 产生 critical | 0 |
| held-out project 单独报告 | 100% |

## 6. 回归策略

每次提交至少运行：

```text
unit
synthetic topology fixtures
legacy golden
new topology golden
first two sets compatibility
validation projects
heldout only at release gate
```

不得在查看 heldout 结果后继续针对该项目调规则，然后仍称其为 heldout。
