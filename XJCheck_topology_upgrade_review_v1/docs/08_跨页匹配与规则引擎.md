# 跨页匹配与规则引擎

## 1. 项目图

项目图至少包含：

```text
Sheet -> Network -> Endpoint -> TerminalIdentity
                       |
                       +-> CrossPageReference
```

跨页匹配不是在所有数字中做笛卡尔积。候选作用域依次缩小：

1. 同一项目；
2. 可审计页面；
3. open endpoint / symbol port / interruption point；
4. terminal/wire identity compatible；
5. reference page/column compatible；
6. direction/medium compatible；
7. reciprocal/global constraints。

## 2. EndpointIdentity

推荐规范化：

```json
{
  "raw": "5FD25",
  "namespace": "terminal",
  "strip": "5FD",
  "local": "25",
  "scope": null,
  "canonical": "terminal:5FD:25"
}
```

不要把 `25` 与 `5FD25` 无条件视为同一个数字。

## 3. 规则执行输入

Rule 只读取：

- asserted networks；
- selected semantic attachments；
- selected cross-page matches；
- unresolved/possible evidence；
- project profile；
- rule scheme。

## 4. Issue 聚类

同一根因可能触发多个表面规则。例如一个错误 bridge 可能造成 one-to-many、duplicate pair 和 overmerge。应按共享 topology decision / network / endpoint 聚类，输出一个主 Issue 和多个 symptoms。

## 5. 规则分层

### 数据完整性

- Reader 不完整；
- 页码冲突；
- sidecar 缺失；
- unknown high-impact symbol。

### 拓扑

- dangling/open；
- overmerge/split；
- ambiguous crossing；
- multiple nets on port。

### 语义

- unlabeled endpoint；
- ambiguous token；
- invalid terminal prefix；
- terminal identity conflict。

### 跨页

- missing reciprocal；
- different counterpart；
- one-to-many；
- many-to-one；
- duplicate reference；
- medium mismatch。

## 6. Issue 解释模板

```text
第 08 页 endpoint E-102 被识别为 terminal:5FD:25，
其网络 N-41 经由 handle 31A/31B 到达 open endpoint。
项目中唯一 reciprocal 候选位于第 14 页，但该端点被识别为 terminal:5FD:26。
两个端点的 topology、symbol port 和文本附着均为高置信，
因此判定为跨页端子身份冲突。
```
