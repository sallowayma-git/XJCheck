# 依赖与数据源清单

## 1. DWG/DXF 后端

### ODA File Converter

用途：批量 DWG↔DXF；Windows/Linux/macOS；CLI、递归和 audit。  
定位：MVP 默认 Reader 前处理。  
改造：支持 Linux AppImage/DEB/RPM，不再硬编码 Windows 路径。

### ODA Drawings SDK

用途：高保真原生 DWG、xdata、复杂对象、渲染。  
定位：后续生产增强。

### Autodesk RealDWG

用途：官方源 DWG/DXF 读写，适合 .NET/C++ Windows 高保真路径。  
定位：后续可选 Reader。

### GNU LibreDWG

用途：开源 DWG 读/DXF/JSON/文本检索；可作为交叉验证。  
风险：官方标记 beta，部分高级 R2010+ 实体可能跳过。

### ezdxf

用途：DXF 实体解析、绘图/SVG、ODA addon。  
保留为 Python 主解析层。

## 2. 几何与图

### Shapely STRtree

空间候选、bbox、相交和最近邻。

### NetworkX

MVP 图分析和解释。若单页原子边规模大、性能成为问题，再评估 rustworkx/igraph；不要过早更换。

### OR-Tools CP-SAT

全局 attachment/port/cross-page candidate 选择。

## 6. 符号参考数据

### 本项目 block corpus

最重要的数据源。来自 502 张 DWG 的 block definitions、instances、external line entry 和 ATTRIB。

### QElectroTech

可参考开源电气符号分类和元素组织方式。不要直接把其图形当当前企业块的真值。

### KiCad symbols

可参考库 schema、别名、pin 和 unit 的组织方式，主要适用于电子原理图，不直接覆盖二次柜图。

### IEC 60617 / GB/T 4728

用于符号 taxonomy、名称和端口含义参考。按实际许可使用，不应把受限数据库直接复制进交付仓库。
